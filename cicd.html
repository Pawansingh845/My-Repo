CI (Continuous Integration) is the practice of frequently merging code to a central repository and automatically testing it.
CD (Continuous Delivery/Deployment) is the practice of deploying code automatically to staging or production after passing all tests.

üß∞ What is Jenkins?

Jenkins is an open-source automation server used to build, test, and deploy applications. It helps implement CI/CD pipelines efficiently.

  • Supports plugins for Git, Docker, Kubernetes, etc.
  • Can trigger jobs via Git commits, webhooks, or schedules
  • Supports scripted and declarative pipelines

üìÑ Sample Jenkins Declarative Pipeline


pipeline {
  agent any

  stages {
    stage('Clone') {
      steps {
        git 'https://github.com/user/project.git'
      }
    }

    stage('Build') {
      steps {
        sh 'npm install'
      }
    }

    stage('Test') {
      steps {
        sh 'npm test'
      }
    }

    stage('Deploy') {
      steps {
        sh 'echo Deploying to production...'
      }
    }
  }
}
